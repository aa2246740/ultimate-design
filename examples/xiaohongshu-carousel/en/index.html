ULTIMATE DESIGN 01 / 09

You can spot the template before you read the message.

That is the problem with many AI design skills: the output converges faster than the thinking.

Same glass panels. Same rounded cards. Same gradient hero. Same “premium” mood. The content changes, but the design behavior does not.

The tell: layout is chosen before message hierarchy, medium, audience, or proof.

THE REAL GAP 02 / 09

Rules prevent bad work. Taste prevents forgettable work.

Rules

Contrast, spacing, hierarchy, responsive behavior, accessibility, export stability.

Taste

What to avoid, when to be quiet, when type should lead, where memory should live.

Content

The user’s meaning, decision path, proof, risks, and the order in which ideas must land.

Proof

Rendered screenshots, visual checks, critique, repair, and a contract others can continue.

Design quality = message fit × visual hierarchy × taste discipline × rendered proof Take one away and the skill becomes either a checklist or a style costume.
Skill anatomyNo single house style
CONTRACT FIRST 03 / 09

Ultimate Design starts with a contract, not a canvas.

DESIGN.md

Request AnchorWhat did the user actually ask for?
Content ModelWhat should people understand, trust, and remember?
Taste SignatureWhich defaults are avoided? What carries visual memory?
Quality GatesWhat must be checked before delivery?

A contract gives the agent a source of truth after the prompt gets long, the artifact gets complex, and taste decisions start competing with clarity.

  • No dependency on the user bringing a design file.
  • Assumptions are explicit, not hidden in the output.
  • The next agent can continue without reverse-engineering the visuals.

CONTENT LEADS 04 / 09

The layout is a reading path, not a decoration layer.

Ultimate Design treats copy, hierarchy, and typography as part of design, not as text poured into boxes later.

01

Rank what matters: primary message, explanation, evidence, risk, action.

02

Choose a path: Z, F, modular, editorial, process, diagram, or dense report mode.

03

Map emphasis: size, position, grouping, spacing, type, contrast, then color.

04

Check the result: can a reader understand it in the platform’s real viewing scene?

TASTE ENGINE 05 / 09

The skill names the lazy defaults before they sneak in.

No equal-card reflexCards are used when modular scanning means something, not as the default container for every idea.
No generic AI glowGradients, glass, and soft panels must earn their job or leave the composition.
No fake proofProduct truth, data, screenshots, diagrams, or explicit placeholders beat decorative simulation.
No one-style trapLayout families vary by narrative job, not by swapping copy inside the same frame.

Taste is operational: design read, dials, anti-default locks, layout-family budget, type posture, critique.

OKF ROUTER 07 / 09

It does not paste a giant textbook into every run.

Brief is vague?

Infer safely, record assumptions, ask only when the wrong choice would invalidate the work.

Social graphic?

Load graphic, content, taste, typography, verification, and only the needed concepts.

Product UI?

Route into product sense, state language, accessibility, responsive behavior, and component rules.

Deck or PPT?

Route into narrative spine, slide archetypes, decision mode, export stability, and slide-level proof.

CRITIQUE + PROOF 08 / 09

The workflow is not “make it, then hope.”

repair pass
Critique against the Request Anchor, not against vague “looks good” taste.
Fix hierarchy drift, cramped spacing, awkward wraps, and AI-default aesthetics.
Render every final frame, then inspect the full set, not one lucky screenshot.
Record what changed so the design can survive another agent and another edit.
Good design leaves evidence: screenshots, checks, critique notes, and a governed contract.
