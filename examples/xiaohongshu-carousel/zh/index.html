ULTIMATE DESIGN01 / 09

你是不是也能一眼看出：这是 AI 设计？

很多设计 Skill 的问题不是“不好看”，而是太容易被识别：像模板，像默认值，像换字。

同样的玻璃卡片、同样的紫蓝渐变、同样的圆角模块、同样的“高级感”。内容变了，设计行为没变。

真正的问题：还没理解内容，就已经选好了版式。

痛点：模板指纹1080 × 1440 / 3:4
核心差距02 / 09

规则只能兜底，品味决定上限。

规则

对比、留白、层级、响应式、可访问性、导出稳定性。

品味

知道该安静还是该出手，知道该避开什么“AI 味”。

内容

先搞清用户要理解什么、相信什么、记住什么。

证据

渲染截图、视觉检查、挑刺修复、留下设计合同。

设计质量 = 内容契合 × 视觉层级 × 品味约束 × 真实验证 少一项，就容易变成“会排版的模板”。
不是只加规则而是给 Agent 加判断
先钉住需求03 / 09

Ultimate Design 不是先画图，而是先写设计合同。

DESIGN.md

Request Anchor用户最初到底要什么？最后有没有跑偏？
Content Model这套图要让人先懂什么，再信什么？
Taste Signature这次避开什么模板感？靠什么形成记忆点？
Quality Gates交付前必须检查什么，而不是靠感觉。

设计合同不是文档仪式感，它是让 Agent 在长任务里不忘本的“锚”。

  • 用户不用自带 DESIGN.md，Agent 会从上下文自己建立。
  • 假设、取舍、风格理由都显式记录，方便追责和迭代。
  • 下一位 Agent 接手时，不需要反向猜设计意图。
Single Source of Truth设计不是一次性截图
内容先行04 / 09

设计不是装饰，是把内容讲明白。

中文图文不能照搬英文页面习惯。标题、断句、信息密度、留白节奏，都要按中文阅读重新组织。

01

先排信息优先级：核心观点、解释、证据、风险、行动。

02

再选阅读路径：封面钩子、对比、流程、清单、图解或总结页。

03

用字号、位置、分组、间距建立层级，颜色只是放大器。

04

最后检查：手机上能不能扫读？有没有一眼看懂的主线？

中文语境重写不是英文直译
反模板机制05 / 09

先命名套路，才有机会摆脱套路。

不默认卡片化只有模块扫描真的有意义时，才用卡片承载内容。
不乱加高级感玻璃、阴影、渐变、光效都要有表达任务。
不制造假证据没有真实截图、数据、图解时，明确留占位，不假装。
不一套版式刷九页每页版式服务叙事角色，而不是换文案。

Taste Engine 的作用，不是规定一种风格，而是约束 Agent 不要滑回默认审美。

Anti-template locks有风格，但不为风格牺牲内容
字体也是设计06 / 09
字

中文排版不能只换字体。

黑体现代、理性、效率感强，适合工具、产品、科技内容。
宋体有编辑感和权威感，适合观点、长文、知识型封面。
圆体亲和、轻松、生活方式，但不适合严肃 B2B 主表达。
等宽适合代码、参数、流程节点，少量使用会增强系统感。
标题可以有性格，正文必须能读。字的“声音”要服务内容，而不是证明设计师很会玩。
Type Personality中文优先，再处理中英文混排
OKF 路由07 / 09

知识不是越多越好，而是用到时刚好出现。

需求模糊？

先推断安全默认值，把假设写进 Request Anchor，只问真正会挡路的问题。

社媒图文？

加载平面图文、内容模型、Taste Engine、字体人格和视觉验证。

产品页面？

进入产品判断、状态语言、可访问性、响应式和组件规则。

PPT / Deck？

进入叙事主线、页面角色、决策目标、导出稳定性和逐页检查。

需求→合同→分支知识→产物
按需索引减少噪声，保留专业判断
挑刺 + 修复08 / 09

不是做完就交付，而是先把低级问题抓出来。

渲染截图
间距扫描
修复复查
拿 Request Anchor 对照：有没有忘记用户最初要的东西？
检查标题、正文、卡片、底栏有没有贴太近、遮挡、错位。
看整套截图，不只看某一页“刚好没出事”的样子。
修完再验证，并把改动写回设计合同。
专业不是不犯错，而是有流程把错抓出来、修掉，再交付。
Rendered proof真实截图优先于理论正确
为什么值得用09 / 09

Ultimate Design：给 AI 加一套设计脑。

当你要做的不是“随便好看一下”，而是页面、PPT、海报、报告、产品 UI、品牌系统这类需要讲清楚、看得懂、能迭代的设计，就该用它。

给 Agent支持 Codex、Claude Code、Pi Agent 等常见 Coding Agent 工作流。
给创作者做落地页、图文、PPT、讲解页、产品界面，不再只靠模板。
给团队留下 DESIGN.md，让后续修改不漂移、不失忆。
ultimate-design skill值得拥有，但更值得被正确使用。
