# Purpose

Use this concept when a visible artifact needs senior design judgment rather than another surface style: public-facing pages, decks, social graphics, brand systems, high-polish product UI, redesign critique, or any result that feels styled, overdecorated, template-like, fake-minimal, or "AI-generated."

This is a judgment lens, not an Apple-style preset. Do not copy Apple visual surfaces, brand assets, layouts, glass effects, icons, product language, or a white-and-rounded house style. Learn the discipline: every visible decision should feel necessary for the user, message, material, medium, and scene.

# Core Model

```text
necessity
-> inevitability
-> hierarchy
-> craft tolerance
-> care
-> material honesty
-> scene fit
```

Taste can create memory, but necessary design creates trust. A strong design should feel like it could not be much else without becoming less clear, less humane, less truthful, or less useful.

# Human Value Lens

Use these eight questions as a cross-check, not an Apple visual preset:

- **Purpose:** Is this worth the user's time, attention, and trust? What was deliberately left out?
- **Agency:** Can the user understand, choose, cancel, reverse, undo, or recover in proportion to the consequence?
- **Responsibility:** Does the design protect privacy, safety, truth, and vulnerable users instead of optimizing only for conversion or spectacle?
- **Familiarity:** Does it build on useful conventions and real-world mappings? When it breaks one, is the improvement tested rather than assumed?
- **Flexibility:** Does it adapt to device, language, ability, expertise, and context without forcing one brittle path?
- **Simplicity:** Is the common path clear without confusing minimal appearance with actual ease of use?
- **Craft:** Are spacing, type, alignment, motion, states, and edge cases deliberate enough to earn trust over repeated use?
- **Delight:** Does the intended feeling emerge from the other seven qualities rather than from decorative confetti or novelty?

Record only the questions that change the artifact. If the lens produces no decision, do not preserve it as ceremonial rationale.

# Trigger

Load this concept when:

- The user asks for taste, design philosophy, "design sense", Apple/Jony Ive/Rams/Hara-style judgment, senior critique, or why a design feels professional.
- The artifact has public-facing polish, meaningful style freedom, or high brand/product trust impact.
- A result looks like a template, card stack, fake minimalism, decorative "premium", or style applied after the content.
- Final critique must decide what to remove, demote, simplify, align, justify, or make more humane.

Do not load it for tiny implementation-only fixes unless the visual decision itself is the problem.

# Operational Tests

Run these tests in critique and polish. Record only the findings that materially change the work.

## Delete Test

Remove or hide the element in imagination. If function, understanding, emotional precision, identity, trust, or navigation does not get worse, remove it or demote it.

## Replace Test

Swap the shape, color, position, type weight, icon, or copy with a generic alternative. If the design is equally strong, the relationship is not yet specific enough.

## Move Test

Move the element slightly. If the composition, reading path, or hierarchy does not weaken, the layout may be loose. Tighten grid, alignment, grouping, or proportion until placement carries intent.

## Justification Test

Every major spacing, radius, shadow, motion, weight, image crop, and color-area ratio should have a reason tied to hierarchy, task, material, interaction, brand, or scene. "It looks premium" is not enough.

## Care Test

Check moments that reveal whether the designer cared: first use, waiting, loading, empty, error, disabled, confirmation, undo, permission, reduced motion, keyboard/focus, long text, and small screens. When relevant, these states need useful copy, calm feedback, and recoverable paths.

## Material Honesty Test

Visuals must not imply false interactivity, false proof, false depth, false product capability, false data certainty, or real assets that do not exist. A mockup, placeholder, generated image, or inferred claim must be labeled or designed as provisional when truth matters.

## Scene Fit Test

Density, quietness, type expression, motion, and decoration must fit the use scene. A dashboard may need quiet density; a campaign poster may need memorable type; a board deck may need proof before flourish; a teaching page may need sequence, examples, and recovery from confusion.

# Critique Vocabulary

Use this pattern for actionable critique:

```text
observation -> principle -> consequence -> repair
```

Examples:

- This floating label competes with the report card; because the label is not necessary for comprehension, it weakens the first reading path. Remove it or merge it into the card heading.
- The page uses large whitespace as a style cue, but the missing evidence makes the claim feel less trustworthy. Add proof or make the promise smaller.
- The animated number is memorable, but it is the only moving object in a dense decision surface. Tie the motion to a data reveal and stop it before the user starts comparing.

Avoid vague feedback such as "make it Apple", "more premium", or "cleaner." Name what is unnecessary, arbitrary, careless, dishonest, or scene-mismatched, then repair it.

# Source Confidence

- Dieter Rams and Vitsœ principles are stable high-confidence source material for restraint, usefulness, honesty, and thoroughness.
- Apple HIG and style guidance are official but current platform details should be verified from current official sources when building platform UI.
- Apple's official *Principles of great design* framework is high-confidence source material for purpose, agency, responsibility, familiarity, flexibility, simplicity, craft, and delight. Use it as human-value judgment, not trade-dress guidance.
- Jony Ive influence, Apple product case studies, and Hara-style readings are useful source notes, but many public accounts are interpretive. Treat them as judgment vocabulary, not binding facts.
- Avoid brand imitation and protected trade dress. This concept teaches decision quality, not Apple appearance.

# Done Check

This concept is applied when another agent can answer:

1. What was removed or demoted because it was not necessary?
2. What must remain for understanding, trust, emotion, identity, or action?
3. Which relationships now feel inevitable rather than arbitrary?
4. Which craft tolerances were tightened: spacing, alignment, radius, type weight, icon weight, motion ending, or image crop?
5. Which care states were designed or explicitly judged irrelevant?
6. What material honesty risk was fixed or recorded?
7. Why does the density and expression fit the scene?
