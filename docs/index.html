01 / WHERE WORK BREAKS

一张页面，常常只完成了一半。

画面做出来以后，真正的问题才露出来：需求在中途变了，内容没讲清，下一位接手的人不知道哪些是刻意选择。

01

需求在途中换了

原始诉求被新细节盖住，最后做出的东西已经不是一开始要解决的问题。

02

重要的话没有位置

内容和版式一起开工，真正需要被看懂的部分反而被装饰和组件挤到后面。

03

交付只剩一张图

没有取舍记录，也没有真实状态的检查，下一位只能从头猜一遍。

02 / WHAT STAYS

每一次设计，都留下能继续使用的东西。

需求变了，看需求卡；换人接手，读 DESIGN.md；出现问题，回到真实页面和截图。

01

Request Anchor

这次要解决什么，谁来用，什么不能丢。

02

DESIGN.md

内容顺序、视觉方向、假设和待确认项写在一起。

03

真实复核

用真实页面和截图看内容、层级、换行和细节是否站得住。

03 / THE RUN

先把事情说清，再把画面做对。

五步会留下五个可回看的点：需求、内容、方向、成品和复核。

  1. 01

    读需求

    保留原始诉求、最新调整和验收标准。

  2. 02

    理内容

    确定先让人看懂什么，哪些话必须准确。

  3. 03

    定方向

    选择合适的色彩、字体、密度、版式和参考策略。

  4. 04

    做成品

    落到网页、产品界面、PPT、图形或品牌材料。

  5. 05

    回看交付

    检查真实输出，修问题，补上下一位能读懂的记录。

有参考图就用它判断结构、气质和节奏；没有也照样继续。它是输入，不是流程的前提。

04 / WHERE IT HELPS

不同媒介，仍然可以用同一套判断。

做的东西不同，看的问题不同；但内容、方向、成品和复核的顺序不需要变。

官网与内容页

把调研、想法或产品说明，整理成读得懂的页面。

产品界面

Dashboard、表单、引导、定价页、流程和原型。

PPT / Deck

汇报、路演、培训、产品讲解和 HTML Deck。

平面与社媒

海报、九图、报告封面、横幅和信息图。

品牌规则

色彩、字体、识别点和跨媒介的使用边界。

审查与打磨

检查已有设计，修掉影响理解和使用的问题。

05 / START HERE

用一句明确的话开始。

普通工作直接说你要做什么。需要共同决定方向时，再加 --pro。

$ ultimate-design

先做一版

$ultimate-design 帮我把这份研究做成一个便于阅读的网页。先处理内容和设计方向，再实现、复查并修掉明显问题。

先对齐关键选择

$ultimate-design --pro
我要做一个面向投资人的产品官网。先和我确定受众、信息重点、品牌气质和验收标准，再开始实现。

06 / HANDOFF

交出去的，不只有一个链接。

工作完成后，下一位人或 Agent 能看到做了什么、为什么这样做，以及还有什么没解决。

  1. 01DESIGN.md需求、取舍、假设和质量门写在同一个可继续的合同里。
  2. 02参考输入参考资料保留来源和用途；没有参考图也会写清楚。
  3. 03真实检查页面、幻灯片或图形按实际尺寸看过，而不是只看代码。
  4. 04未解决风险没有解决的限制和待确认项如实留下。

从一个明确的任务开始，后面的取舍也有据可查。
